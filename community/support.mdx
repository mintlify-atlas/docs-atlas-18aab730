## Getting Help

### Before Asking for Help

Before reaching out for support:

1. **Check the Wiki** - Search the [wiki documentation](https://wiki.idempiere.org) for existing guides and solutions
2. **Search the Forums** - Look through [forum discussions](https://groups.google.com/group/idempiere) to see if your question has been answered
3. **Try the Test Sites** - Reproduce your issue on the [live test sites](https://www.idempiere.org/test-sites) to verify behavior
4. **Check JIRA** - Search the [issue tracker](https://idempiere.atlassian.net/issues/) for known bugs or feature requests

### Where to Ask Questions

<Steps>
  <Step title="Start with Forums or Mattermost">
    For general questions, support requests, and discussions, use:
    - [Google Groups Forum](https://groups.google.com/group/idempiere) for detailed questions and async discussions
    - [Mattermost Support Channel](https://mattermost.idempiere.org/idempiere/channels/support) for real-time help during business hours
  </Step>
  
  <Step title="Submit Bug Reports to JIRA">
    After confirming a bug through discussion:
    - Submit a ticket to [JIRA](https://idempiere.atlassian.net)
    - Include clear reproduction steps
    - Provide relevant logs and screenshots
  </Step>
  
  <Step title="Follow Community Guidelines">
    - Be respectful and follow the [Code of Conduct](https://github.com/idempiere/idempiere/blob/master/CODE_OF_CONDUCT.md)
    - Provide detailed information about your issue
    - Share relevant system details and versions
  </Step>
</Steps>

## Reporting Bugs

### Before Submitting a Bug Report

<AccordionGroup>
  <Accordion title="Reproduce in Latest Version">
    Verify the issue exists in the latest version of iDempiere. Many bugs are fixed in newer releases.
  </Accordion>
  
  <Accordion title="Check Existing Issues">
    Search [JIRA](https://idempiere.atlassian.net/issues/) to ensure the bug hasn't already been reported. If you find a closed issue that matches your problem, open a new issue and reference the closed one.
  </Accordion>
  
  <Accordion title="Test on Test Sites">
    If possible, reproduce the bug on one of the [official test sites](https://www.idempiere.org/test-sites) before creating a ticket.
  </Accordion>
  
  <Accordion title="Discuss First">
    Share your findings in the forums or Mattermost to get initial feedback and confirmation from the community.
  </Accordion>
</AccordionGroup>

### How to Submit a Good Bug Report

When creating a bug report in JIRA:

- **Use a clear, descriptive title** that identifies the problem
- **Provide exact reproduction steps** with as much detail as possible
- **Include specific examples** - ideally reproducible with Garden World data
- **Describe observed vs expected behavior** - explain what happened and what should have happened
- **Add screenshots or videos** showing the problem
- **Include stack traces** from log files if the application crashed
- **Provide performance metrics** if reporting performance issues
- **Specify your environment** - iDempiere version, database, operating system

<Warning>
  **Security Vulnerabilities**: Do not open JIRA tickets for security vulnerabilities. Instead, follow the [security policy](https://wiki.idempiere.org/en/How_to_report_a_vulnerability).
</Warning>

## Requesting Features

### Before Submitting an Enhancement

1. **Discuss in the Forums** - Suggest your change in the [public forum](https://groups.google.com/group/idempiere) and collect feedback before opening a JIRA ticket
2. **Search for Existing Requests** - Check if a similar enhancement has already been suggested
3. **Consider Plugins** - Evaluate if your enhancement is better suited as a [community plugin](https://wiki.idempiere.org/en/Category:Available_Plugins)

### Creating an Enhancement Request

When submitting an enhancement to JIRA:

- **Clear title** describing the enhancement
- **Step-by-step description** of the suggested improvement
- **Specific examples** demonstrating the use case
- **Current vs desired behavior** - explain what exists and what you want
- **Explain the value** - why this would benefit most iDempiere users
- **Justify core vs plugin** - why it should be in core rather than a plugin
